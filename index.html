Légende Couleurs

CORE - Tables fondamentales (auth, membres, audit)
FIN - Finance & Comptabilité (OHADA/SYSCOA)
OPS - Opérations & Chaîne Valeur (collecte, stock, vente)
GOV - Gouvernance & Légal (AG, documents, audit)

Exemple SQL: Table Membres

CREATE TABLE membres ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID NOT NULL REFERENCES app_users(id), numero_membre VARCHAR(50) UNIQUE NOT NULL, nom_complet VARCHAR(255) NOT NULL, telephone VARCHAR(20), region VARCHAR(100), date_adhesion DATE NOT NULL DEFAULT CURRENT_DATE, statut ENUM('actif', 'suspendu', 'inactif') DEFAULT 'actif', genre ENUM('M', 'F', 'Autre'), age INTEGER, nce_nir VARCHAR(20), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); CREATE INDEX idx_membre_region ON membres(region); CREATE INDEX idx_membre_statut ON membres(statut);

Principales Relations

app_users → membres
Chaque utilisateur a un profil membre (ou staff)
membres → comptes_epargne
Un membre = un compte épargne avec solde + intérêts
membres → demandes_credit
Un membre peut avoir plusieurs prêts actifs
recoltes → lots_produits → ventes
Farm-to-table traceability. Chaque vente = lot identifié.
journal_comptable → bilan_comptable
Bilan auto-généré du journal (OHADA).
assemblees → presences_ag
Suivi quorum et signatures numériques.

✓ Points Clés du Schéma

Conformité OHADA/SYSCOA: Plan comptable 5-niveaux, journal débits/crédits, bilan auto-généré, compte de résultat automatisé.
Traçabilité Complète: Lot-tracking (recolte → stock → vente), QR codes, parcelle GPS, intrants utilisés, certifications.
Audit Trail: Chaque transaction + horodatage + responsable. Immutable pour compliance.
Inclusion & ESG: Genre/age (monitoring), tenure security, ESG indicators readiness.
Scalabilité: 52 tables, 280+ colonnes, index optimisés. Prêt pour 5000+ membres.
Sécurité: RLS (Row-Level Security) par rôle. Encryption ready. Audit trail immutable.
